# Tailscale DNS Conflict — Troubleshooting Guide

When Tailscale is installed on a Raspberry Pi, it takes control of DNS configuration. This can cause package installation and general internet name resolution to fail. This guide explains why it happens and how to fix it.

---

## Understanding the Problem

Tailscale manages `/etc/resolv.conf` to route DNS queries through its own resolver (`100.100.100.100`). This enables features like MagicDNS — resolving other Tailscale devices by hostname. However, Tailscale's DNS resolver may not reliably forward public domain queries depending on your network configuration.

The file itself warns against manual edits:
```
# DO NOT EDIT THIS FILE BY HAND -- CHANGES WILL BE OVERWRITTEN
```

Any changes you make are reverted immediately by Tailscale's daemon.

---

## Symptoms

Commands like `apt update` or `apt install` fail with:

```
Temporary failure resolving 'deb.debian.org'
```

Internet connectivity works fine (you can ping IP addresses directly), but domain name resolution fails.

---

## Diagnosis

### Confirm Internet Works

```bash
ping -c 3 8.8.8.8
```

If this succeeds, your internet connection is fine — the problem is DNS only.

### Confirm DNS Is Broken

```bash
ping -c 2 deb.debian.org
```

If this fails with `Temporary failure in name resolution`, DNS is the issue.

### Check Current DNS Config

```bash
cat /etc/resolv.conf
```

If you see `nameserver 100.100.100.100`, Tailscale is managing DNS.

---

## Solution

### Step 1: Disable Tailscale DNS Management

```bash
sudo tailscale set --accept-dns=false
```

This tells Tailscale to stop overwriting `/etc/resolv.conf`. The setting persists across reboots.

### Step 2: Set Public DNS Servers

```bash
echo -e "nameserver 8.8.8.8\nnameserver 1.1.1.1" | sudo tee /etc/resolv.conf
```

This sets Google DNS as primary and Cloudflare DNS as fallback.

### Step 3: Verify

```bash
ping -c 2 deb.debian.org
```

You should now get successful responses.

---

## Impact on Tailscale

Disabling DNS management does **not** break Tailscale VPN. All tunneling and routing continues to work normally.

| Feature | Status after fix |
|---------|-----------------|
| SSH via Tailscale IP | ✅ Works |
| Tailscale routing and tunneling | ✅ Works |
| MagicDNS (hostname resolution) | ❌ No longer works |
| SSH via short device hostname | ❌ No longer works |

### Workaround for MagicDNS

If you rely on connecting to devices by hostname rather than IP, add entries to `/etc/hosts`:

```bash
sudo nano /etc/hosts
```

Add a line for each device:
```
100.x.x.x    your-device-name
```

---

## Prevention

On any new Pi installation with Tailscale, run these commands immediately after joining the Tailscale network — before installing anything else:

```bash
sudo tailscale set --accept-dns=false
echo -e "nameserver 8.8.8.8\nnameserver 1.1.1.1" | sudo tee /etc/resolv.conf
```
